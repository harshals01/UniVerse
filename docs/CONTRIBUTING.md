# Contributing to UniVerse

First off, thank you for considering contributing to UniVerse! It's people like you that make UniVerse such a great platform for the campus community.

## Where do I go from here?

If you've noticed a bug or have a feature request, make one! It's generally best if you get confirmation of your bug or approval for your feature request this way before starting to code.

## Fork & create a branch

If this is something you think you can fix, then fork UniVerse and create a branch with a descriptive name.

## Get the test suite running

Make sure you're able to run the local dev server and test suite. See the main README for installation instructions.

## Implement your fix or feature

At this point, you're ready to make your changes! Feel free to ask for help if you need it.

## Make a Pull Request

At this point, you should switch back to your master branch and make sure it's up to date with UniVerse's master branch.
Then create a Pull Request outlining your changes.
